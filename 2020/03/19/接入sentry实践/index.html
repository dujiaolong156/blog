    接入步骤

    1.去 [Sentry] 官网注册账号，新建项目

    2.准备项目分支 feature/sentry

    3.安装 SDK

    4.配置 SDK, 将sentry集成到项目中

  console.log(nam);
// 手动抛出一个 Error

    6.检测异常通知是否收到，若收到通知，表示项目接入 Sentry 初步完成
在测试出发 event 之后，查看两处是否收到异常通知
- 注册的邮箱中

- Sentry 的 Issue 列表页

7.增加项目成员，在项目所在 team 中增加开发同学的邮箱，这样大家都可以接收到异常监控通知

## 接入后已拥有的能力
- 监控到异常后第一时间通知到邮箱(后期会把开发小伙伴的邮箱都加入,目前只有世博和我)
- 定位到出错的 url
- 具体的错误文本、类型
- 客户端的一些信息（浏览器版本、操作系统、用户行为）
- 区分develop 和 production 环境
- TODO:  定位到出错的代码文件（配合 sourceMap ）
- TODO:  添加 release 信息，如何解决每次发版自动添加 release 信息
## 关于前端监控方面正在尝试的事
- 接入 [FullStory](https://www.fullstory.com/), 在异常发生时支持回放用户行为，当前进行到 Sentry 账号与 FullStory 账号绑定阶段
## 一些采坑记录
1. 想自己尝试接入流程的同学，避免使用QQ邮箱注册 Sentry ，会提示邮箱格式不合法
1. 项目中一些自定义标签在浏览器报错的信息，是不被 Sentry 收集的
## Sentry 还可以做什么
- 过滤错误信息，添加 ignoreErrors, 可以将那些不需要被关注的异常信息过滤掉，使其不在通知范围内
- 定制错误信息，通过 setExtra、setUser、setTag 等方法定制错误信息
## tips
关于上传 sourceMap，我们只有在开发环境，才会用到 sourceMap，线上环境如果加载sourceMap，不仅影响用户体验，而且也不安全，那 Sentry 是如何解决的？Sentry 会要求用户把 sourceMap 上传到 Sentry 的服务器，通过每次的 release 版本号来管理 sourceMap，如果上传到 Sentry 官网的服务器觉得不安全外，可以自建sentry服务。官方提供了两种方式
    - sentry-cli 命令行
    - webpack插件

    >官方文档：https://docs.sentry.io/cli/configuration/#properties-files
